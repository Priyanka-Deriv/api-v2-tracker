1. Overview & Purpose

This document summarises the full build conversation for the Deriv API V2 Live Team Tracker — a shared, real-time project management tool built inside Claude to help Priyanka and her cross-functional team manage the V1→V2 API migration at Deriv.

The tracker was built iteratively over one conversation, evolving from a basic Kanban board into a fully-featured team operations hub with AI-powered transcript parsing, Slack integration, team scratchpads, and live shared storage.

💡 Core problem solved: The team works at high speed across 17+ teams with action items emerging from meetings, Slack messages, and stakeholder reviews constantly. No existing tool was lightweight enough to keep up — so a custom live tracker was built directly in Claude.

2. Features Built

📋
Kanban Board
5 columns: Backlog → In Progress → Blocked → In Review → Done. Move cards with one tap.
👥
4 Views
Kanban, By Person, By Type, Team Scratchpad — switch instantly to see what you need.
⚡
Transcript Parsing
Paste any meeting transcript → Claude auto-extracts action items, assigns owners, sets priorities.
✏️
Inline Editing
Click any title or notes field directly on a card to edit. No modal needed for quick fixes.
🔀
Merge Mode
Select 2+ cards and merge duplicates. Pick primary title, notes combine, highest priority wins.
🗒️
Team Scratchpad
Per-team workspace with Open Questions, Decisions, Dependencies (bidirectional), To-dos, Notes.
💬
Slack Sync (Read)
Reads from #project_deriv_api_v2 and #task_api_v2_website. Auto-creates action items from messages mentioning team members or tasks.
🌐
Live Shared Storage
All changes persist and sync across all team members in real-time. No refresh needed.
⚠️
Duplicate Detection
When Slack sync finds a task that may already exist, it flags it for review rather than silently duplicating.
🏷️
Item Types
Action Item, Product Decision, Review, Bug/Fix, Documentation, External Request, Stakeholder Input.

3. Key Design Decisions Made

Q: How should items be tagged to teams?
✅ One primary owner + list of dependent teams. Each item has one team that owns execution, plus any teams that need to act or are impacted.
Q: Which views matter most day-to-day?
✅ All four: Kanban by status, By Person, By Type, and Team Scratchpad.
Q: For external teams (SEO, Data, Marketing) — what to track?
✅ Request log, current status, POC per team, and whether they're blocked by us or blocking us.
Q: Which fields go wrong most often from transcript parsing?
✅ Task title and Notes/context — both now have prominent inline editing with auto-select on click.
Q: When should Slack sync run?
✅ Auto on open + manual ↻ Sync Slack button.
Q: What Slack activity creates an action item?
✅ Messages mentioning a team member (@name) OR messages that look like tasks/to-dos.
Q: If a task already exists, what happens on Slack sync?
✅ Flag as possible duplicate for review — never silently create duplicates.
